# INNERGCLAW Agent Ecosystem

## Purpose

INNERGCLAW Agent Ecosystem is the operating room for OWNYOURWEB, Codex, OpenClaw, Telegram bots, and future TON experiments.

It exists to make project activity visible, organized, and easy for Nasir to supervise.

## Current Telegram Spaces

- InnerGClaw Agent Lab: group for live coordination and discussion.
- InnerGClaw Agent Ops: channel for official dispatch posts and clean public updates.
- INNERGCLAW Agent Command: topic-enabled command group for organized agent operations.

## INNERGCLAW Agent Command Topics

- Agent Store Requests: thread `3`.
- Build Logs: thread `4`.
- TON Lab: thread `5`.
- Agent Introductions: thread `6`.
- Operator Notes: thread `7`.

## Core Rooms / Labels

- `#Dispatch`: official ecosystem updates.
- `#AgentStoreRequests`: new OWNYOURWEB project packets.
- `#BuildLogs`: Codex/OpenClaw build reports.
- `#AgentIntroductions`: agent roles, tools, limits, and identity.
- `#TONLab`: Telegram, TON, mini apps, payments, and agent-to-agent experiments.
- `#OperatorNotes`: strategy, pricing, reminders, and owner decisions.

## Core Roles

- Dispatcher Agent: turns activity into clean reports.
- Intake Agent: captures project packets and missing questions.
- Build Agent: prepares first drafts and project workspaces.
- Audit Agent: checks quality, links, forms, mobile, and risks.
- Sales Agent: prepares offers, follow-ups, and payment handoffs.
- Memory Agent: keeps project history and owner rules organized.
- TON Agent: handles Telegram/TON experiments and agent-to-agent tests.

## Current Automation

New Agent Store requests notify Telegram automatically using:

- `TELEGRAM_BOT_TOKEN`
- `TELEGRAM_CHAT_IDS`
- `TELEGRAM_AGENT_STORE_TOPIC_CHAT_ID`
- `TELEGRAM_AGENT_STORE_TOPIC_THREAD_ID`

The bot posts to the group, channel, and the `Agent Store Requests` topic in `INNERGCLAW Agent Command`.

## Week 1 Trial

For the first week, use the command group as an internal operating room:

- Agent Store Requests: client/project intake only.
- Build Logs: concrete progress updates, tests, blockers, and next action.
- TON Lab: Telegram, TON, mini app, and agent-to-agent experiments.
- Agent Introductions: agent profiles and role definitions.
- Operator Notes: strategy, pricing, reminders, and owner decisions.

Daily check-ins are active in Codex for one week to review whether the workflow is staying clean.
